<summary>:shipit: Pull Request Checklist and Guidelines</summary>

### Creating the Pull Request

- [ ] Write a [perfect pull request](https://github.blog/2015-01-21-how-to-write-the-perfect-pull-request/#approach-to-writing-a-pull-request)
  :vulcan_salute: Short and descriptive title
  :yarn: Explain **why** you made the changes (thought process, technical reasons, etc.)
  :thinking: Call out any specific areas that need assistance or extra scrutiny
- [ ] Set Reviewers and Assignees
    - Reviewers: Who needs to review the code before it can proceed
    - Assignee: Who should currently be looking at this pull request
- [ ] If this pull request is tied to a release, set the Milestone to the release version
- [ ] Set [needs labels](https://github.com/intelisys/myintelisys/labels?q=needs%3A) for any special requirements for the changes to go live


### After Creating the Pull Request

- [ ] Ensure the Jira card is transitioned to "Code Review"
- [ ] Ensure [needs labels](https://github.com/intelisys/myintelisys/labels?q=needs%3A) match special requirements
- [ ] Create testing notes!


### Code Review Guidelines

- It's a conversation, here are some guidelines for:
  :speech_balloon: [Offering feedback](https://github.blog/2015-01-21-how-to-write-the-perfect-pull-request/#offering-feedback)
  :love_letter: [Responding to feedback](https://github.blog/2015-01-21-how-to-write-the-perfect-pull-request/#responding-to-feedback)
- If you can, check out the branch and run the code
- Change the Assignee to indicate who should be looking at the pull request at the moment
